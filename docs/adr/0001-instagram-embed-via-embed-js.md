# Instagram post embed via official embed.js

The "Destaque no Instagram" section in `PlayerOverlayCard` embeds a specific player post using Instagram's official `<blockquote class="instagram-media">` + `embed.js` approach, loaded lazily on first expansion. We chose this over a static thumbnail preview (which would require a server-side proxy to fetch Instagram's OEmbed API) and over `<iframe>` (blocked by Instagram's `X-Frame-Options`). The `embed.js` script is injected once into `document.head` and subsequent calls reuse it via `window.instgrm.Embeds.process()`. A direct `"Abrir no Instagram"` button is always rendered below the embed as a fallback, so the user is never stranded if the script fails to load.

This blockquote + `embed.js` mechanism is encapsulated in the shared `src/components/InstagramEmbed.tsx` component (with the `window.instgrm` type and the one-time script-injection guard living there, and the permalink safety check in `src/utils/instagram.ts`). It is reused by three call sites: the player overlay card, the official FIFA reel on the Redes Sociais tab, and the "Destaques no Instagram" feed (`InstagramHighlightsFeed`, fed by `src/data/instagramHighlights.ts`). The feed keeps the same lazy posture deliberately — only the first card mounts its embed by default and the rest mount on tap — so a feed of many highlights never spins up many Instagram iframes at once.

## Addendum — iframe variant for the per-match Instagram tab

The original "blocked by `X-Frame-Options`" finding applies to the **canonical post URL** (`/p/<code>/`, which sends `X-Frame-Options: DENY`). Instagram's **dedicated `/embed/` endpoint** (`/p/<code>/embed/`) is a different story: it ships *without* `X-Frame-Options` and without a `frame-ancestors` CSP, precisely so it can be framed — it is exactly the iframe `embed.js` builds internally.

The empty-`blockquote` + `embed.js` approach has one practical weakness: it renders nothing until `embed.js` loads and hydrates it, and `embed.js` is a frequent target of content/tracker blockers (uBlock, Brave, privacy extensions). When the script is blocked, the user sees a blank embed and only the fallback link.

So the per-match Instagram tab (the "Instagram" tab on the Ao Vivo match page, `MatchDetailView`, fed by `src/data/matchInstagram.json`) uses a script-free `<iframe>` straight to the `/embed/` endpoint instead — `src/components/InstagramPostFrame.tsx`, with the permalink→embed-URL conversion (`toInstagramEmbedUrl`) in `src/utils/instagram.ts`. It listens for Instagram's `MEASURE` `postMessage` to size the iframe to the post's content height, and keeps the same always-present "Abrir no Instagram" fallback link. The three original call sites are unchanged — they keep the `embed.js` blockquote (and their e2e tests assert on that markup).
